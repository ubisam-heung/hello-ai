## 화면 구성 요소 설명

### 1. 상단 앱 바
- **로고 및 제목**: "Hello DB - Multi-Database CRUD"
- **파란색 배경**: Material Design primary 컬러
- **데이터베이스 아이콘**: mdi-database

### 2. 데이터베이스 선택 카드
```
┌─────────────────────────────────────────────┐
│ 데이터베이스 선택          ●  연결됨          │
│                                              │
│ 🗄️ [MySQL                            ▼]    │
│                                              │
│ 선택 가능한 옵션:                            │
│   - MySQL                                    │
│   - MariaDB                                  │
│   - PostgreSQL                               │
│   - Oracle                                   │
└─────────────────────────────────────────────┘
```
- **위치**: 화면 상단
- **기능**: 
  - 데이터베이스 유형 선택
  - 선택 시 자동으로 해당 DB의 사용자 목록 로드
  - 녹색 연결 상태 표시

### 3. 사용자 추가 폼 (왼쪽)
```
┌────────────────────────────┐
│ 사용자 추가                │
│                            │
│ 👤 이름                    │
│ [홍길동____________]        │
│                            │
│ 📧 이메일                  │
│ [hong@example.com___]      │
│                            │
│ [➕ 추가하기]              │
└────────────────────────────┘
```
- **위치**: 화면 왼쪽 (4칼럼)
- **기능**:
  - 이름 입력 필드 (필수)
  - 이메일 입력 필드 (필수, 형식 검증)
  - 추가 버튼 (primary 컬러)
  - 폼 유효성 검사

### 4. 수정 모드 (왼쪽)
```
┌────────────────────────────┐
│ 사용자 수정                │
│                            │
│ 👤 이름                    │
│ [김철수____________]        │
│                            │
│ 📧 이메일                  │
│ [kim@example.com____]      │
│                            │
│ [💾 수정] [취소]           │
└────────────────────────────┘
```
- 수정 버튼 클릭 시 폼이 수정 모드로 전환
- 기존 데이터가 폼에 자동 입력
- 저장 및 취소 버튼 표시

### 5. 사용자 목록 테이블 (오른쪽)
```
┌─────────────────────────────────────────────┐
│ 사용자 목록 (5)                     🔄      │
│                                              │
│ ┌────┬──────┬──────────────┬─────────┬───┐ │
│ │ ID │ 이름 │    이메일    │ 생성일  │   │ │
│ ├────┼──────┼──────────────┼─────────┼───┤ │
│ │ 5  │홍길동│hong@exam.com │방금 전  │✏️🗑│ │
│ │ 4  │김철수│kim@exam.com  │2분 전   │✏️🗑│ │
│ │ 3  │이영희│lee@exam.com  │5분 전   │✏️🗑│ │
│ │ 2  │박민수│park@exam.com │1시간 전 │✏️🗑│ │
│ │ 1  │정수진│jung@exam.com │3시간 전 │✏️🗑│ │
│ └────┴──────┴──────────────┴─────────┴───┘ │
└─────────────────────────────────────────────┘
```
- **위치**: 화면 오른쪽 (8칼럼)
- **컬럼**:
  - ID (자동 생성)
  - 이름
  - 이메일
  - 생성일 (한국어 상대 시간)
  - 작업 버튼
- **기능**:
  - ✏️ 수정 버튼 (파란색)
  - 🗑 삭제 버튼 (빨간색)
  - 🔄 새로고침 버튼

### 6. 빈 목록 상태
```
┌─────────────────────────────────────────────┐
│ 사용자 목록 (0)                     🔄      │
│                                              │
│  ℹ️  등록된 사용자가 없습니다.              │
│     새 사용자를 추가해보세요!                │
└─────────────────────────────────────────────┘
```

### 7. DB 선택 안 된 상태
```
┌─────────────────────────────────────────────┐
│ 사용자 목록                         🔄      │
│                                              │
│  ℹ️  데이터베이스를 먼저 선택하세요.        │
└─────────────────────────────────────────────┘
```

### 8. 스낵바 (하단)
```
                ┌──────────────────────────────┐
                │ ✅ 사용자가 추가되었습니다!  │
                │                      [닫기]  │
                └──────────────────────────────┘
```
- **위치**: 화면 하단
- **종류**:
  - 성공 (녹색): "사용자가 추가되었습니다"
  - 성공 (녹색): "사용자가 수정되었습니다"
  - 성공 (녹색): "사용자가 삭제되었습니다"
  - 에러 (빨간색): "데이터 로드 실패: [에러 메시지]"
- **기능**: 3초 후 자동 닫힘, 수동 닫기 버튼

## 사용자 인터랙션 흐름

### 시나리오 1: 사용자 추가
```
1. 데이터베이스 선택 ─→ [MySQL ▼]
                     │
                     ▼
2. 이름 입력 ───────→ [홍길동]
                     │
                     ▼
3. 이메일 입력 ─────→ [hong@example.com]
                     │
                     ▼
4. 추가 버튼 클릭 ──→ [➕ 추가하기]
                     │
                     ▼
5. 서버 요청 ───────→ POST /api/mysql/users
                     │
                     ▼
6. 성공 응답 ───────→ { id: 5, name: "홍길동", ... }
                     │
                     ▼
7. 목록 새로고침 ───→ [테이블 업데이트]
                     │
                     ▼
8. 스낵바 표시 ─────→ [✅ 사용자가 추가되었습니다!]
                     │
                     ▼
9. 폼 초기화 ───────→ [입력 필드 비우기]
```

### 시나리오 2: 사용자 수정
```
1. 수정 버튼 클릭 ──→ [✏️] (테이블 행)
                     │
                     ▼
2. 폼에 데이터 로드 → [이름: 홍길동, 이메일: hong@...]
                     │
                     ▼
3. 정보 수정 ───────→ [이름: 김철수]
                     │
                     ▼
4. 수정 버튼 클릭 ──→ [💾 수정]
                     │
                     ▼
5. 서버 요청 ───────→ PUT /api/mysql/users/5
                     │
                     ▼
6. 성공 응답 ───────→ { id: 5, name: "김철수", ... }
                     │
                     ▼
7. 목록 새로고침 ───→ [테이블 업데이트]
                     │
                     ▼
8. 스낵바 표시 ─────→ [✅ 사용자가 수정되었습니다!]
                     │
                     ▼
9. 폼 초기화 ───────→ [수정 모드 해제]
```

### 시나리오 3: 사용자 삭제
```
1. 삭제 버튼 클릭 ──→ [🗑] (테이블 행)
                     │
                     ▼
2. 확인 대화상자 ───→ [정말 삭제하시겠습니까?]
                     │
                     ├─[취소]→ 종료
                     │
                     ▼
                   [확인]
                     │
                     ▼
3. 서버 요청 ───────→ DELETE /api/mysql/users/5
                     │
                     ▼
4. 성공 응답 ───────→ { message: "User deleted" }
                     │
                     ▼
5. 목록 새로고침 ───→ [테이블에서 행 제거]
                     │
                     ▼
6. 스낵바 표시 ─────→ [✅ 사용자가 삭제되었습니다!]
```

### 시나리오 4: 데이터베이스 전환
```
1. DB 선택 ─────────→ [MySQL ▼]
                     │
                     ▼
2. MySQL 데이터 표시 → [ID: 1, 2, 3, 4, 5]
                     │
                     ▼
3. DB 변경 ─────────→ [PostgreSQL ▼]
                     │
                     ▼
4. 목록 자동 로드 ──→ GET /api/postgresql/users
                     │
                     ▼
5. PostgreSQL 데이터 → [ID: 1, 2, 3]
   (독립적인 데이터)
```

## 컬러 스킴 (Material Design)

- **Primary**: #1976D2 (파란색) - 버튼, 앱 바
- **Secondary**: #424242 (어두운 회색) - 보조 요소
- **Success**: #4CAF50 (녹색) - 성공 메시지, 연결 상태
- **Error**: #FF5252 (빨간색) - 에러 메시지, 삭제 버튼
- **Info**: #2196F3 (밝은 파란색) - 정보 메시지
- **Warning**: #FFC107 (노란색) - 경고 메시지

## 반응형 디자인

### 데스크톱 (>960px)
- 폼: 왼쪽 4칼럼
- 테이블: 오른쪽 8칼럼
- 나란히 배치

### 태블릿 (600-960px)
- 폼: 상단 전체 폭
- 테이블: 하단 전체 폭
- 세로 배치

### 모바일 (<600px)
- 모든 요소 전체 폭
- 세로 배치
- 테이블 스크롤 가능

이 UI는 직관적이고 사용하기 쉽게 설계되었습니다!
